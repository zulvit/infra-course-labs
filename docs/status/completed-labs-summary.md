## Подтверждение по лабораторным

### Лабораторная работа 1

Отчёт `lab1/report.md` прямо фиксирует полный результат:
ВМ поднимается через `Vagrant`, приложение отвечает на `http://127.0.0.1:8080`,
после проверки стенд удаляется.

### Лабораторная работа 2

Отчёт `lab2/report.md` фиксирует полный multi-node сценарий:
подняты `front`, `back`, `db`, выполнены запросы к frontend и backend,
данные добавлены и считаны из PostgreSQL.

### Лабораторная работа 3

В `lab3/report.md` и `lab3/final_report.md` зафиксированы:
загрузка образа, запуск контейнера, bind mount, HTTP-проверка и сохранение образа в архив.

### Лабораторная работа 4

Работа была пересобрана и перепроверена в нормальном Docker-окружении.
Подтверждены:

- сборка и запуск образа `hello`;
- сборка и запуск образа `main` с проверкой `ARG/ENV/RUN/COPY/ADD/EXPOSE/LABEL/ENTRYPOINT/CMD`;
- сборка и запуск multi-stage образа `multi`;
- запуск `registry:2`, `push` образа и проверка Registry HTTP API.

Актуальные материалы:

- `lab4/report.md`
- `lab4/final_report.md`
- `lab4/artifacts/logs/`

### Лабораторные работы 5-8
