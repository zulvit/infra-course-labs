## Цель

Отработать базовый цикл с контейнерным окружением: скачать образ, запустить контейнер, проверить версию интерпретатора, поднять простой HTTP-сервер с примонтированным каталогом, сохранить образ в архив.

## Ход работы

Системного Docker с демоном в среде не оказалось (`docker` / `docker info` не дали рабочего варианта без прав администратора). Практику провёл через **udocker** в пользовательском каталоге (установка pip-пакетов в `$HOME`), чтобы не требовать root.

Примеры команд:

```bash
udocker pull python:3.11.5-alpine
udocker run --rm python:3.11.5-alpine python --version
```

Проверка ответа веб-сервера с хоста (порт в сценарии — 8888):

```bash
wget -qO- http://127.0.0.1:8888
```

Сохранение образа в tar выполнялось через `udocker save` (см. `artifacts/logs/step04_udocker_save.typescript`).

Полный сценарий с перехватом вывода лежит в `artifacts/logs/session_lab3.typescript`.
