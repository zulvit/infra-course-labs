# Лабораторная работа 3

## Тема

Основы работы с контейнерами

## Цель работы

Изучить базовые операции работы с контейнерами: загрузку образа, запуск контейнера, выполнение команды внутри контейнера, использование bind mount и сохранение образа в архив.

## Ход выполнения

### 1. Подготовка среды

На начальном этапе было проверено наличие Docker в системе.  
Нативный Docker отсутствовал, поэтому для выполнения лабораторной в пользовательском режиме был использован `udocker`.

Подготовленные файлы:

- `work/test.py`
- `work/run.sh`

### 2. Загрузка образа и запуск контейнера

Через `udocker` был загружен образ `python:3.11.5-alpine`.  
После этого был выполнен запуск контейнера с командой проверки версии Python.

Результат:

- образ был успешно загружен;
- контейнер запускался корректно;
- внутри контейнера была подтверждена работа Python.

Фрагмент вывода:

```text
executing: python
Python 3.11.5
```

### 3. Создание контейнера и работа с bind mount

Был создан именованный контейнер, затем подготовлен сценарий запуска HTTP-сервера внутри контейнера с подключением рабочего каталога через bind mount.

Результат:

- каталог проекта успешно подключался внутрь контейнера;
- контейнер запускал HTTP-сервис;
- с хоста был получен корректный HTTP-ответ.

### 4. Проверка ответа приложения

После запуска HTTP-сервера был выполнен запрос к локальному порту.

Результат:

- приложение возвращало ожидаемый ответ;
- работа контейнера и проброса данных была подтверждена практически.

Полученный HTTP-ответ:

```text
Hello from lab3 via udocker.
```

### 5. Просмотр информации об образе и контейнере

Дополнительно были выполнены команды просмотра информации о контейнере и образе.

Результат:

- получены метаданные контейнера;
- проверен manifest образа;
- подтверждён корректный импорт образа в пользовательскую среду `udocker`.

### 6. Сохранение образа

На завершающем этапе образ был сохранён в tar-архив.

Результат:

- образ успешно сохранён в файл;
- архив может использоваться как артефакт лабораторной работы.

Фрагмент вывода:

```text
-rw-rw-r-- 1 savva savva 19M Apr  9 02:30 ../artifacts/logs/python-3.11.5-alpine.tar
```

## Результаты

В ходе лабораторной работы были успешно выполнены и проверены:

- загрузка образа;
- запуск контейнера;
- выполнение команды внутри контейнера;
- создание именованного контейнера;
- использование bind mount;
- запуск HTTP-приложения внутри контейнера;
- сохранение образа в tar-архив.

## Вывод

В результате выполнения лабораторной работы были изучены основные операции работы с контейнерами.  
Несмотря на отсутствие системного Docker, поставленные задачи были воспроизведены в пользовательском режиме с помощью `udocker`, что позволило подтвердить базовые сценарии жизненного цикла контейнера на практике.
