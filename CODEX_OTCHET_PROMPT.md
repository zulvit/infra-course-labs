# Промпт для Codex: финальный отчёт + DOCX через otchet-compose

Используй **после** того, как выполнены (или честно задокументированы) лабораторные `lab1`–`lab8`. Задача — **сжать** сырые логи и разрозненные `report.md` в **один связный отчёт** и сгенерировать **Word (DOCX)** утилитой [otchet-compose](https://github.com/sorivma/otchet-compose).

---

## 0. Утилита otchet-compose (локально, не в git)

В корне проекта (рядом с папками `lab1`, `lab2`, …) должна лежать клонированная копия:

```bash
git clone https://github.com/sorivma/otchet-compose.git otchet-compose
```

Каталог `otchet-compose/` **не коммитится** (уже в `.gitignore`).

Установка и запуск (из корня проекта):

```bash
cd otchet-compose
pip install -e .
cd ..
```

Проверка:

```bash
otchet-compose gen -f path/to/config.yml
```

Если конфиг лежит в текущей директории как `otchet-compose.yml`, достаточно:

```bash
otchet-compose gen
```

Итоговый файл создаётся по пути `document.output` из YAML; каталоги создаются автоматически.

Официальные подсказки по формату YAML для нейросети лежат в репозитории утилиты:

- `otchet-compose/doc/prompt.txt` — правила для LLM;
- `otchet-compose/doc/otchet-compose.llm.ref.yml` — развёрнутый эталон с комментариями;
- `otchet-compose/examples/otchet-compose.yml` — рабочий пример.

## 1. Что прочитать из проекта (источники правды)

Для каждой лабораторной `labN`:

1. **`labN/task.md`** — тема, ожидаемый результат, смысл работы (что *должно* было быть по курсу).
2. **`labN/report.md`** — что сделано, блокеры, итог.
3. **`labN/artifacts/logs/`** — **не копировать целиком**. Выбери только:
   - команды, которые реально отражают ход работы;
   - короткие фрагменты stdout/stderr, подтверждающие успех или характер ошибки;
   - итоговые проверки (`curl`, `docker ps`, `vagrant status`, и т.д.).

Игнорируй:

- повторяющиеся попытки одной и той же команды;
- длинные логи сборок, если в отчёте достаточно факта «образ собран / сервис запущен»;
- всё, что не относится к формулировкам из `task.md`.

При наличии скриншотов: `labN/assets/screenshots/*.png` — можно сослаться в блоках `figure` (путь относительно YAML, см. ниже).

## 2. Структура итогового документа (содержательно)

Собери **один** DOCX на весь курс **или** отдельный DOCX на каждую лабу — по указанию пользователя. По умолчанию: **один сводный отчёт** `build/Final_DevOps_Infra_Report.docx`.

Содержание сводного отчёта:

1. **Введение и постановка** (внутри раздела «Цель работы» первыми абзацами): зачем цикл лаб, какие инструменты (Vagrant, Docker, Compose, Ansible), на какой среде выполнялось (кратко: ОС, WSL, ограничения виртуализации — только если влияло на результат).
2. **Кратко, что делали по лабораторным** — по одному сжатому абзацу или маркированному списку на лабу, строго по смыслу `task.md`, без воды.
3. **Ход работы** — для **каждой** лабы подзаголовок уровня 2 (`level: 2`, `structural: false`):
   - краткая последовательность шагов (3–7 пунктов, не пересказ логов);
   - **таблица** с колонками вроде: «Этап / Команда (или группа команд) / Результат» — только репрезентативные команды, не 50 строк;
   - при блокере — отдельный короткий абзац: причина, что сделано взамен (если было), ссылка на формулировку из `report.md`.
4. **Заключение**: что освоено, что подтверждено практикой, что не удалось воспроизвести в среде и почему (если применимо).

Никаких «простыней» из логов в теле отчёта.

## 3. Формат YAML для otchet-compose (строго)

Соблюдай схему из `otchet-compose/doc/prompt.txt` и пример `examples/otchet-compose.yml`:

- `version: 1`
- `document.output` — например `./build/Final_DevOps_Infra_Report.docx` (относительно каталога, откуда запускаешь `otchet-compose gen`, лучше из **корня проекта**).
- `document.toc: true`
- Титульный лист: `document.title_page` с шаблоном `rut` или `mock` (см. README otchet-compose). Заполни `params` **реальными** ФИО, группой, преподавателем, дисциплиной, номером работы (для сводного отчёта — например «по лабораторным работам 1–8» в `work_title` / `discipline` по смыслу). Если данных нет — оставь однозначные плейсхолдеры `TODO_...`, чтобы пользователь заменил перед сдачей.
- При использовании `title_page` **не** ставь `reserve_title_page: true` (как в примерах).

Обязательные структурные разделы (`level: 1`, `structural: true`):

- «Цель работы» — сюда же логически входит введение + формулировка целей цикла лаб.
- «Ход работы» — все лабы, таблицы, списки, при необходимости рисунки.
- «Заключение».

Внутри «Хода работы» — только подзаголовки `level: 2`, `structural: false` (например «Лабораторная работа 1. Vagrant», …).

Типы блоков **только** поддерживаемые: `heading`, `paragraph`, `figure`, `list`, `table`.

- Текст абзацев — в `paragraph.text` в виде folded scalar `>`.
- Списки — `type: list`, `style: bullet` или `numeric`.
- Подпись рисунка **без** номера («Рисунок 1» добавит генератор). В тексте перед рисунком можно отослаться к нему словами «на рисунке ниже».
- `figure.path` — путь **относительно YAML-файла**. Если конфиг лежит в корне проекта, укажи например `./lab5/assets/screenshots/step01.png`. Если файла нет — блок `figure` без `path` (будет placeholder).

## 4. Выходные артефакты, которые ты создаёшь

1. Файл конфигурации, например: **`otchet-compose.final.yml`** в корне репозитория с лабораторными (рядом с `lab1`…`lab8`).
2. Запуск генерации:

   ```bash
   otchet-compose gen -f otchet-compose.final.yml
   ```

3. Краткая запись в **`FINAL_DOCX_README.md`** (необязательно, по желанию пользователя): путь к YAML, путь к DOCX, какие плейсхолдеры в титуле нужно заменить и повторно запустить `gen`.

---

## 5. Ограничения и аккуратность

- Не выдумывай выполнение шагов, которых нет в `report.md` / логах.
- Не включай секреты (токены, пароли из `.env`).
- Сохраняй нейтральный академический стиль, без разговорных оборотов.
- Если пользователь просит **отдельный DOCX на каждую лабу**, создай `labN/otchet-compose.yml` и `document.output: "./labN/build/labN.docx"` для каждой, с титулом под номер лабы.

## Краткая шпаргалка команд для пользователя (после твоей работы)

```bash
# из корня проекта (где lab1, lab2, … и папка otchet-compose)
pip install -e ./otchet-compose
otchet-compose gen -f otchet-compose.final.yml
```

Открыть готовый файл из `document.output` в Microsoft Word / LibreOffice и при необходимости вручную поправить титульный лист.
