# Лабораторная работа 5

## Тема

Docker Volumes and Networks

## Цель работы

Изучить работу Docker с томами и сетями, а также проверить способы хранения данных и сетевого взаимодействия между контейнерами.

## Ход выполнения

### 1. Подготовка среды

На рабочей машине был установлен Docker и проверена его работоспособность.  
После этого были подготовлены рабочие файлы лабораторной:

- `work/run.sh`
- `work/index.html`

### 2. Проверка bind mount

Был подготовлен HTML-файл на хостовой машине и подключён в контейнер `nginx` через bind mount.  
Контейнер был запущен с публикацией порта на хост.

Результат:

- страница успешно открывалась с хоста;
- содержимое файла с хоста корректно отображалось в контейнере.

Полученный ответ:

```html
<html>
  <body>
    <h1>lab5 bind mount</h1>
  </body>
</html>
```

### 3. Проверка named volume

Были созданы именованные тома Docker.  
Один из томов использовался как источник данных для контейнера `nginx`.

Результат:

- данные из тома успешно использовались контейнером;
- веб-сервер возвращал ожидаемый HTML-ответ.

Полученный ответ:

```html
<html><body><h1>lab5 named volume</h1></body></html>
```

### 4. Совместное использование тома

Были запущены два контейнера, использующие один и тот же том:

- один контейнер записывал данные;
- второй контейнер читал эти данные.

Результат:

- общий файл был доступен из обоих контейнеров;
- это подтвердило корректную работу shared volume.

Полученный вывод:

```text
shared-from-writer
```

### 5. Проверка сетей Docker

Сначала была проверена работа контейнеров в стандартной сети `bridge`.  
После этого была создана пользовательская bridge-сеть с собственной подсетью.

Результат:

- в стандартной сети имя контейнера не резолвилось автоматически;
- в пользовательской сети контейнеры успешно взаимодействовали друг с другом по имени;
- сетевое соединение работало корректно.

Фрагмент вывода ping:

```text
PING lab5-second (10.0.0.3): 56 data bytes
64 bytes from 10.0.0.3: seq=0 ttl=64 time=0.050 ms

--- lab5-second ping statistics ---
1 packets transmitted, 1 packets received, 0% packet loss
```

### 6. Проверка режима host network

Контейнер был запущен в режиме `host`, внутри него был поднят HTTP-сервис.

Результат:

- сервис был доступен напрямую через сетевой стек хоста;
- дополнительная публикация порта не требовалась.

Фрагмент ответа сервиса:

```html
<title>Directory listing for /</title>
<h1>Directory listing for /</h1>
```

## Результаты

В ходе лабораторной работы были успешно проверены:

- bind mount;
- named volume;
- совместное использование тома между контейнерами;
- стандартная и пользовательская bridge-сеть;
- режим `host`.

## Вывод

В результате выполнения лабораторной работы были изучены основные механизмы хранения данных и сетевого взаимодействия Docker.  
Практически подтверждена работа bind mount, named volumes, общей работы с томами, пользовательских сетей и режима `host`.
