# Лабораторная работа 4

## Тема

Сборка контейнерных образов и работа с локальным registry

## Цель работы

Изучить сборку Docker-образов по `Dockerfile`, публикацию образов в локальный registry и проверку полученного результата.

## Ход выполнения

### 1. Подготовка среды

Для выполнения лабораторной в пользовательском режиме были использованы:

- `udocker` для запуска контейнеров без системного Docker daemon;
- `kaniko` для сборки образов по `Dockerfile`.

Были подготовлены рабочие каталоги и примеры Dockerfile:

- `work/hello/Dockerfile`
- `work/main/Dockerfile`
- `work/main/main.py`
- `work/multi/Dockerfile`
- `work/multi/main.go`
- `work/run.sh`

### 2. Сборка простого образа

С помощью `kaniko` был собран минимальный образ `hello`.  
Сборка выполнялась без Docker daemon с сохранением результата в tar-архив.

Результат:

- сборка простого образа завершилась успешно;
- был получен артефакт `hello.tar`.

### 3. Запуск локального registry

Через `udocker` был запущен локальный контейнер `registry:2`.

Результат:

- registry успешно стартовал;
- его HTTP API отвечал корректно;
- работоспособность registry была подтверждена запросом к `/v2/`.

Подтверждение из логов:

```text
listening on [::]:5000
GET /v2/ HTTP/1.1 200
```

### 4. Публикация образа в registry

После успешной сборки образ `hello:0.1` был отправлен в локальный registry.

Результат:

- push завершился успешно;
- registry содержал опубликованный образ;
- через HTTP API были подтверждены наличие репозитория и список тегов.

Фрагмент вывода:

```text
Pushed 127.0.0.1:5000/hello@sha256:7ea46a9d31cbc64c0f310c33a4b519bc08b3ef58f75da6308a82f3c47d088671
{"repositories":["hello"]}
{"name":"hello","tags":["0.1"]}
```

### 5. Проверка более сложных Dockerfile

Дополнительно были выполнены попытки собрать более сложные образы:

- Python-приложение;
- multi-stage образ на Go.

Результат:

- сборка этих вариантов в пользовательской среде завершалась ошибками на операциях распаковки rootfs и `chown`;
- тем самым было подтверждено ограничение текущей среды для полноценной сборки сложных образов без системного Docker.

Фрагменты ошибок:

```text
error building image: error building stage: failed to get filesystem from image: chown /bin: operation not permitted
error building image: error building stage: failed to get filesystem from image: chown /boot: operation not permitted
```

### 6. Проверка возможности запуска собранных образов

Была выполнена попытка импортировать результат сборки в `udocker` для последующего запуска.

Результат:

- файловая система образа импортировалась;
- однако runtime metadata образа не сохранялась полностью;
- поэтому корректный запуск собранных образов в полном виде в текущей среде не воспроизводился.

## Результаты

В ходе лабораторной работы были успешно подтверждены:

- сборка простого контейнерного образа по `Dockerfile`;
- сохранение результата сборки в tar-архив;
- запуск локального registry;
- публикация образа в registry;
- проверка registry через HTTP API.

Дополнительно были выявлены ограничения пользовательской среды при сборке и запуске более сложных образов.

## Вывод

В результате выполнения лабораторной работы была изучена сборка контейнерных образов и работа с локальным registry.  
Практически подтверждена возможность сборки и публикации простого образа в условиях пользовательской среды.  
Также были установлены ограничения такого режима работы при попытке собирать и запускать более сложные образы.
