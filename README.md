# DevOps Study Workspace

## Что находится в корне

В корне репозитория находятся основные учебные разделы и инструмент генерации отчётов:

- `lab1`-`lab8` — исходные лабораторные работы курса;
- `РХД` — отдельный блок работ по PostgreSQL;
- `Go` — задания по Go;
- `ПКП` — архитектурные кейсы;
- `otchet-compose` — локальный генератор DOCX-отчётов.

## Как устроены работы

Каждая оформленная работа хранится по одному и тому же принципу:

- `report.md` — человекочитаемый отчёт;
- `otchet-compose.yml` — источник для генерации DOCX;
- `build/` — итоговые DOCX-файлы;
- `artifacts/` — вспомогательные логи и промежуточные артефакты, если они нужны для воспроизводимости.

## Документация для Codex

Все служебные инструкции и промпты вынесены в каталог `docs/`:

- `docs/codex/prompts/` — промпты для запуска и продолжения работы;
- `docs/codex/guides/` — памятки по генерации отчётов;
- `docs/status/` — сводные служебные документы по состоянию проекта.

## Базовые данные для титульных листов

Во всех актуальных отчётах используется единый набор метаданных:

- студент: `Купленик С. С.`;
- группа: `УВП-171`;
- преподаватель: `Старший преподаватель кафедры ЦТУТП Заманов Е. А.`.
