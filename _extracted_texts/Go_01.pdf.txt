Введение
Почему возник Go, рефлексия
• В 2007 году в Google накопилась реальная инженерная боль:
• C++ - огромное время компиляции, опасная работа с памятью
• Java - многословие, JVM overhead, сложная модель 
конкурентности
• Python - медленный, динамическая типизация скрывает 
ошибки до рантайма
• Публичный анонс - 10 ноября 2009 года. Go 1.0 - март 2012. 
• С тех пор - полная обратная совместимость. 
• Текущая стабильная версия - Go 1.26

Три принципа Go
• Простота - это дизайн-решение, не упущение. Нет 
наследования. Нет перегрузки операторов. Нет исключений. 
Нет неявных преобразований типов. Каждое «нет» -
осознанный выбор в пользу читаемости и предсказуемости.
• Производительность через предсказуемость. Компиляция в 
нативный код. Нет виртуальной машины. Программист должен 
понимать, что происходит в памяти.
• Конкурентность - примитив языка, не библиотека. Горутины и 
каналы встроены в язык, а не добавлены поверх.

package
• Каждый .go файл обязан начинаться с package <имя>
• Все файлы в одной директории - один пакет
• Имя пакета ≠ путь импорта (но обычно совпадает с именем 
директории)
• Только package main + func main() = исполняемая программа
• Всё остальное - библиотека

Экспортируемость
• В Go нет ключевых слов public / private. 
• Правило одно - заглавная буква = экспортируется (видна 
вне пакета).
package mypackage
// Экспортируемые - видны из других пакетов
var MaxSize = 100
func Calculate() int { return 42 }
type Config struct { ... }
// Неэкспортируемые - только внутри пакета
var internalCache = map[string]string{}
func validateInput(s string) bool { ... }
type internalState struct { ... }
