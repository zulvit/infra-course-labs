## Цель работы

Цель работы состояла в загрузке справочных данных в PostgreSQL из CSV-файлов и проверке обновления статистики таблиц после выполнения `ANALYZE`. Требовалось автоматически развернуть схему магазина, импортировать данные командой `COPY` и убедиться, что они доступны для чтения и анализа.

## Ход работы

Был подготовлен контейнер PostgreSQL и набор CSV-файлов для таблиц `manufacturers`, `categories`, `stores`, `customers`. В SQL-скрипте использованы команды `COPY ... FROM` для загрузки данных из каталога, смонтированного в контейнер как `/data`.

После импорта выполнена выборка производителей. Ответ PostgreSQL показал реальные строки:

```text
manufacturer_id | manufacturer_name
1               | Apple Inc.
2               | Samsung Electronics
3               | Sony Group Corporation
4               | Lenovo Group
```

Затем была выполнена команда `ANALYZE`, после которой проверено состояние статистики в `pg_stat_user_tables`. В результате для загруженных таблиц появились значения `last_analyze`, а оценка количества строк показала, что статистика была обновлена.

## Основные команды

Запуск стенда:

```powershell
docker compose up -d
```

Проверка данных после `COPY`:

```powershell
docker exec postgres_lab2 psql -U shop_admin -d shop_db -c "SET search_path TO shop; TABLE manufacturers;"
```

Проверка статистики:

```powershell
docker exec postgres_lab2 psql -U shop_admin -d shop_db -c "SELECT relname AS table_name, n_live_tup AS row_count, last_analyze FROM pg_stat_user_tables ORDER BY relname;"
```

Завершение работы:

```powershell
docker compose down -v
```
