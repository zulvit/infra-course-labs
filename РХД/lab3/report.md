## Цель работы

Цель работы состояла в освоении логического резервного копирования PostgreSQL и проверке корректного восстановления базы данных из резервной копии. Требовалось сформировать дампы в custom и plain SQL форматах, восстановить базу в отдельный экземпляр и убедиться, что после восстановления сохраняются данные и ограничения.

## Ход работы

Для работы был развёрнут контейнер PostgreSQL со схемой магазина и тестовыми справочниками. После запуска стенда выполнено логическое резервное копирование базы `shop_db` двумя способами: в формате `custom` и в формате обычного SQL-скрипта.

Затем была создана новая база `shop_db_restored`, в которую выполнено восстановление из custom-дампа. После восстановления проверено количество строк в ключевых таблицах. Ответ PostgreSQL показал следующие результаты:

```text
tbl           | count
manufacturers | 4
categories    | 4
stores        | 3
customers     | 3
```

Дополнительно была выполнена проверка ограничений и связей. В ответе присутствовали типы `PRIMARY KEY`, `UNIQUE`, `FOREIGN KEY`, `CHECK`, что подтверждает перенос не только данных, но и структуры.

## Основные команды

Создание резервной копии:

```powershell
docker exec postgres_lab3 pg_dump -U shop_admin -d shop_db --format=custom -f /backups/shop_backup.dump
```

Восстановление в отдельную базу:

```powershell
docker exec postgres_lab3 pg_restore -U shop_admin -d shop_db_restored /backups/shop_backup.dump
```

Проверка количества строк:

```powershell
docker exec postgres_lab3 psql -U shop_admin -d shop_db_restored -c "SET search_path TO shop; SELECT 'manufacturers' AS tbl, COUNT(*) FROM manufacturers UNION ALL SELECT 'categories', COUNT(*) FROM categories UNION ALL SELECT 'stores', COUNT(*) FROM stores UNION ALL SELECT 'customers', COUNT(*) FROM customers;"
```

- восстановление в отдельную базу прошло успешно;
