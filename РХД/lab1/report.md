## Цель работы

Цель лабораторной работы состояла в развёртывании PostgreSQL в контейнере Docker и создании структуры базы данных магазина с помощью DDL-скрипта. Требовалось подготовить схему `shop`, описать таблицы предметной области, задать ключи, ограничения целостности и проверить, что структура корректно создаётся при инициализации контейнера.

## Ход работы

Для выполнения работы был подготовлен отдельный стенд на основе `docker compose` и официального образа `postgres:16`. В переменных окружения были заданы имя базы данных, пользователь и пароль администратора. В каталог инициализации помещён SQL-скрипт, автоматически выполняемый при первом старте контейнера.

В DDL-скрипте были созданы таблицы `manufacturers`, `categories`, `products`, `stores`, `customers`, `purchases`, `purchase_items`, `deliveries`, `price_change`. Для таблиц заданы первичные и внешние ключи, ограничения `UNIQUE`, а также проверки на корректность количества и цены.

После запуска стенда выполнена проверка списка таблиц в схеме `shop`. Команда запроса к системному каталогу показала, что в базе данных доступны следующие отношения:

```text
schemaname | tablename
shop       | categories
shop       | customers
shop       | deliveries
shop       | manufacturers
shop       | price_change
shop       | products
shop       | purchase_items
shop       | purchases
shop       | stores
```

Таким образом, инициализация контейнера и выполнение DDL прошли успешно, а итоговое число таблиц составило 9.

## Основные команды

Подготовка и запуск стенда:

```powershell
docker compose up -d
```

Проверка состава схемы:

```powershell
docker exec postgres_lab1 psql -U shop_admin -d shop_db -c "SET search_path TO shop; SELECT schemaname, tablename FROM pg_tables WHERE schemaname='shop' ORDER BY tablename;"
```

Остановка стенда:

```powershell
docker compose down -v
```
